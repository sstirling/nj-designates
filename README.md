# Hereby Designated

A data journalism project cataloguing every ceremonial **"Designates X as"** bill introduced in the New Jersey Legislature since 2000 — state symbols, named bridges, commemorative weeks, honorary highway signs. The site is tongue-in-cheek about a genuine quirk of Trenton politics: the amount of the legislative day spent making things official.

The live site runs the full pipeline below end-to-end and renders the archive as an interactive, filterable surface. Data and rule changes are committed to this repo so every number on the site is traceable. It can be [found here.](https://sstirling.github.io/nj-designates/site/) 

Reporting by Stephen Stirling. Data from the [NJ Legislature's bill search](https://www.njleg.state.nj.us/bill-search). Full methodology is in [`docs/METHODOLOGY.md`](docs/METHODOLOGY.md) and on the live site at [`/methodology.html`](site/methodology.html).

## What's in the archive right now

- **3,180 ceremonial bills** across **14 biennial sessions (2000–2027)**.
- **315 became law** (9.9% pass rate across the full archive).
- Primary categories: holiday/observance, state symbol, road naming, place naming, other ceremonial.
- Sponsor leaderboard topped by Angela V. McKnight (194 primary sponsorships since entering the legislature).

These numbers are regenerated by the build step from `site/data/meta.json`. Nothing is hand-typed.

## Quickstart

Requires Python 3.11+.

```bash
git clone https://github.com/sstirling/nj-designates.git
cd nj-designates
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Run the test suite.
pytest

# Serve the already-built site locally.
python -m http.server --directory site 8000
# Open http://localhost:8000
```

The last command serves the site using the pre-built JSON committed in `site/data/`. You only need to run the scraper if you want to refresh the data.

## Refreshing the data

The scraper talks to `njleg.state.nj.us/api/...` — an undocumented but public JSON API the legislature's own bill-search page uses. It is rate-limited to one request per second with jitter; a full refresh across every session takes roughly an hour.

```bash
# Pull one session from the live API.
python -m scraper fetch --session 2024

# Build the processed dataset + site JSON.
python -m scraper build --session 2024

# Or do both for every session in one go. Takes ~60 minutes.
python -m scraper refresh --all

# Regenerate the social share card from the fresh numbers.
python scripts/make_social_card.py
```

After a refresh, `git diff data/processed/bills.parquet` and `git diff site/data/meta.json` show exactly what changed. Commit those (and `site/assets/social-card.png` if regenerated) so the published site matches what's in the repo.

## Refining the filter

The rules that decide what counts as ceremonial live in plain-text files:

- **`data/reference/category_rules.yml`** — include, exclude, and category patterns. Regex, case-insensitive. Editing this file and running `python -m scraper build --all` re-runs the whole pipeline against the cached raw data (no network calls).
- **`data/reference/overrides.yml`** — per-bill `force_include` / `force_exclude` / `force_category` decisions. Use these sparingly; every override should have a one-line comment saying *why*.
- **`data/reference/status_codes.csv`** — hand-curated decoder for the legislature's opaque status codes. Unknown codes are surfaced to readers as "Meaning not documented" rather than invented labels.

After every rule change, run the test suite — the fixtures in `tests/fixtures/` catch regressions where a known-ceremonial bill drops out or a known-admin bill slips in.

## Deploying to GitHub Pages

Settings → Pages → **Deploy from a branch** → `main` / `/site`.

Push to `main` and the site updates in about 60 seconds. Social-card and OG-image URLs in the HTML head assume the default `https://sstirling.github.io/nj-designates/` path. If you set a custom domain, update those six meta-tag URLs in `site/index.html` and `site/methodology.html`.

## Contributing and corrections

Found a bill that should be included and isn't? A category that reads wrong? A sponsor attribution that needs updating? Open an issue on this repository with:

1. The bill's session and number (e.g. `2024-A444`).
2. The live NJ Legislature link for that bill.
3. What you think should change, and why.

Rule changes are welcome as pull requests; please update `docs/audit_log.md` in the same PR so the reasoning is preserved.
